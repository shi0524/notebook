
# 贪心算法

## 两种字符G B 交换

```
一个数组中只有两种字符'G'和'B’，
想让所有的G都放在左侧，所有的B都放在右侧
但是只能在相邻字符之间进行交换操作，
返回至少需要交换几次

思路:
    小贪心：
        同样的字符，顺序不变换
        第1个 G 放在 0 位置
        第2个 G 放在 1 位置
        第i个 G 放在 i-1 位置

    伪代码：
        ans = 0
        index = 0
        for i in str:
            if str[i] == 'G':
               ans += i - index
               index += 1 
讲解:
    大厂刷题班: 第1节 第4题 00:41:55
```

## 难度-工作-报酬
```
给定数组hard和money，长度都为N
hard[i]表示i号的难度， money[i]表示i号工作的收入
给定数组ability，长度都为M，ability[j]表示j号人的能力
每一号工作，都可以提供无数的岗位，难度和收入都一样
但是人的能力必须>=这份工作的难度，才能上班
返回一个长度为M的数组ans，ans[j]表示j号人能获得的最好收入

思路:
    小贪心：
        同样难度选报酬高的   ->  按(难度, 报酬)排序
        不同难度不同报酬去除难度高报酬低的    ->  按(难度, 报酬)排序, 删除难度高，报酬低的
        建立单调性   ->  单调有序数组的二分查找
    细节：
        现在 数组中垫一个 (难度0,报酬0)工作
    讲解:
        大厂刷题班: 第2节 第1题 00:03:30
```

## 过河问题
```
给定一个正数数组arr，代表若干人的体重
再给定一个正数limit，表示所有船共同拥有的载重量
每艘船最多坐两人，且不能超过载重
想让所有的人同时过河，并且用最好的分配方法让船尽量少
返回最少的船数

思路:
    小贪心：
        排序、载重半数位置 L <= half  R > half
        判断左边 L + R 是否大于 limit
        大于：
            L 左滑, 划过位置 打叉
        小于：
            R 右滑 并 计数 p，直到划不动为止
            左边从L位置起，向左拿出 p 个 配成 p 对
        最后：
            左侧先耗尽，右侧剩余  rNum
            右侧先耗尽，左侧剩余全打叉
        
        ans = math.ceil(打叉数/2) + 匹配数 + 右侧剩余数
            
    讲解:
        大厂刷题班: 第3节 第5题 01:39:30
```
